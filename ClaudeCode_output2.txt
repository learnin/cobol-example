     詳細設計

     1. HogeDate.java — DD の COMP-3 対応

     COBOL 定義:
     05 DD PIC 9(2) COMP-3.

     Java 表現方針:
     - dd を int 型で保持（COMP-3 の数値セマンティクスを再現）
     - getDd(): String.format("%02d", dd) で2桁ゼロパディング文字列を返す
     - setDd(String v): 文字列を trim して parseInt → int に変換
     - toCobolString(): yyyy + mm + getDd() （8文字の表示用文字列）
     - fromCobolString(String s): 8文字文字列から s.substring(6,8) をパースして dd に代入

     注記: 実際の COBOL では COMP-3 PIC 9(2) は1バイト(packed decimal)のため
     HOGE-DATE のバイト長は7。集団項目間の MOVE はバイト転送だが、
     Java シミュレーションでは表示用8文字表現で統一し、この点をコメントに明示する。

     MOVE CORR の挙動変化:
     - 旧: DD of YYYYMMDD-R (PIC X(2)) → DD of HOGE-DATE (PIC X(2)) … 文字列コピー
     - 新: DD of YYYYMMDD-R (PIC X(2)) → DD of HOGE-DATE (PIC 9(2) COMP-3) … setDd() を呼ぶことで数値変換

     2. Sub1.java — SUB-WORK-AREA の追加

     // 01 SUB-WORK-AREA — WORKING-STORAGE（プログラム固有）

     3. Sample1.java — ファイル出力の追加

     FILE SECTION のフィールド:

     // FD OUT-FILE / 01 OUT-FILE-REC PIC X(99)

     outFileSec() メソッド:
     private void outFileSec() {
         // DISPLAY "OUT-FILE: " WORK-AREA2
         CobolUtil.display("OUT-FILE: ", workArea2.toCobolString());
         // OPEN OUTPUT OUT-FILE / MOVE WORK-AREA2 TO OUT-FILE-REC / WRITE / CLOSE
         outFileRec = CobolUtil.move(workArea2.toCobolString(), 99);
         try (java.io.PrintWriter pw = new java.io.PrintWriter(
                  new java.io.FileWriter("OUTFILE.TXT"))) {
             pw.println(outFileRec);
         } catch (java.io.IOException e) {
             throw new RuntimeException("FILE OPEN/WRITE error: OUT-FILE", e);
         }
     }

     run() に追加:
     // PERFORM OUT-FILE-SEC  ← PERFORM COPY-SEC の直後に追加
     outFileSec();
