     COBOL → Java 変換プラン: sample1.cbl / sub1.cbl                                                                                                  
   
     Context                                                                                                                                          
                                                                                                                                                    
     sample1.cbl（メインプログラム）・sub1.cbl（サブプログラム）・copy1.cpy（コピーブック）からなる COBOL プログラム群を、等価な Java
     コードへ変換する。
     目的は COBOL 特有の言語機能（REDEFINES、IS EXTERNAL、MOVE CORRESPONDING、CALL USING CONTENT 等）を Java でどう表現するかを明確にすること。

     ビルドツール: なし（javac のみ）
     テスト: main() 実行で出力を目視確認

     COBOL → Java 言語機能マッピング

     ┌─────────────────────────────────┬─────────────────────────────────────────────┐
     │              COBOL              │                    Java                     │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ PIC X(n)                        │ String（CobolUtil.move() で幅固定）         │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ 01 GROUP                        │ クラス                                      │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ REDEFINES                       │ 共有 char[] + getter/setter スライス        │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ IS EXTERNAL                     │ ExternalArea.INSTANCE（static singleton）   │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ CALL "SUB1" USING param         │ Sub1.call(paramDate) — オブジェクト参照渡し │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ CALL "SUB1" USING CONTENT param │ コピーを作成してから Sub1.call(copy)        │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ MOVE CORR A TO B                │ 同名フィールドを静的に列挙して代入          │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ COPY copybook REPLACING         │ クラス名・フィールド名に置換適用済みで展開  │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ DD OF HOGE-DATE OF WORK-AREA2   │ workArea2.hogeDate.getDD()                  │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ PERFORM section                 │ section() メソッド呼び出し                  │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ DISPLAY "text" var              │ CobolUtil.display("text", var)              │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ STOP RUN                        │ run() から return（main() が終了）          │
     ├─────────────────────────────────┼─────────────────────────────────────────────┤
     │ EXIT PROGRAM                    │ call() メソッドから return                  │
     └─────────────────────────────────┴─────────────────────────────────────────────┘

     CobolUtil.java

     ExternalArea.java

     public class ExternalArea {
         public static final ExternalArea INSTANCE = new ExternalArea();
         private String extWorkData = "          "; // PIC X(10)
         // getter / setter（move で幅保持）
     }
     Sample1・Sub1 両方が ExternalArea.INSTANCE を参照 → IS EXTERNAL の共有セマンティクスを再現。

     HogeDate.java

     03 HOGE-DATE（WORK-AREA2 内）と SUB1 の PARAM-DATE は同じ構造なので共通クラスとして定義。
     private String yyyy, mm, dd;  // PIC X(4), X(2), X(2)
     String toCobolString()          // yyyy + mm + dd
     void fromCobolString(String s)  // 8文字からスライス

     WorkArea1.java（REDEFINES の核心）

     private final char[] buf = new char[8]; // YYYYMMDD と YYYYMMDD-R が共有

     // YYYYMMDD: フラットビュー
     String getYYYYMMDD() / void setYYYYMMDD(String)

     // YYYYMMDD-R: 構造ビュー（同じ buf をスライス）
     String getYYYY() / void setYYYY(String)  // buf[0..3]
     String getMM()   / void setMM(String)    // buf[4..5]
     String getDD()   / void setDD(String)    // buf[6..7]
     setMM("03") 後に getYYYYMMDD() が "20260302" を返す → COBOL の REDEFINES 同等。

     WorkArea2.java

     public final HogeDate hogeDate = new HogeDate();
     String toCobolString() { return hogeDate.toCobolString(); }
     void fromCobolString(String s) { hogeDate.fromCobolString(s); }

     CopyWorkArea1.java / CopyWorkArea2.java

     - CopyWorkArea1: copyWorkData PIC X(10)
     - CopyWorkArea2: barStr1/2/3 PIC X(10)（元は FOO-STR1/2/3、REPLACING 適用済み）
     クラスの Javadoc に元のコピーブック名と REPLACING 条件を記録。

     Sub1.java

     public static void call(HogeDate paramDate) {
         // DISPLAY / MOVE "2027" TO YYYY / EXT-WORK-AREA 操作
     }
     パラメータは HogeDate オブジェクト参照。USING CONTENT の場合はコール側でコピーを渡す。

     Sample1.java

     public class Sample1 {
         private final WorkArea1     workArea1    = new WorkArea1();
         private final WorkArea2     workArea2    = new WorkArea2();
         private final CopyWorkArea1 copyWorkArea1 = new CopyWorkArea1();
         private final CopyWorkArea2 copyWorkArea2 = new CopyWorkArea2();
         private static final ExternalArea extWorkArea = ExternalArea.INSTANCE;

         public void run() { moveGrpSec(); redefinesSec(); moveCorrSec(); callSec(); copySec(); ... }
         private void moveGrpSec()   { ... }
         private void redefinesSec() { ... }
         private void moveCorrSec()  { // MOVE CORR: 3フィールドを静的列挙でコピー }
         private void callSec()      {
             // by-reference: Sub1.call(workArea2.hogeDate)
             // by-value:     HogeDate copy = ...; Sub1.call(copy);
         }
         private void copySec()      { ... }

         public static void main(String[] args) { new Sample1().run(); }
     }

     実装順序

     1. CobolUtil.java — 依存なし。move() と display() を実装
     2. ExternalArea.java — CobolUtil のみ依存
     3. HogeDate.java — CobolUtil のみ依存
     4. WorkArea1.java — char[] バッキングストアで REDEFINES 実装
     5. WorkArea2.java — HogeDate に依存
     6. CopyWorkArea1.java / CopyWorkArea2.java — 独立
     7. Sub1.java — HogeDate, ExternalArea, CobolUtil に依存
     8. Sample1.java — 全クラスに依存
     9. run.sh — javac + java を実行するスクリプト
